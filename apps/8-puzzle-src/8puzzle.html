About

This Program:

This is a javascript implementation of the A* Search algorithm to solve an 8 puzzle. A* Search takes 2 things into consideration in order to arrive at a solution: the heurstics and cost of each move. In this implementation, the manhattan distance is used to calculate the heuristic value of each node, and the cost is simply the number of moves needed to arrive to a paticular node. The code for this project can be viewed here.

8 Puzzle Overview:

The 8 puzzle is a 3x3 grid of tiles with 8 of its 9 tiles occupied. This leaves a "free space" (represented by the number 0 in this implementation) that one can manipulate in order to move around and shuffle the other tiles. The objective is to manipulate the tiles of the puzzle until it matches the tiles of the "goal." There are 9!/2 (181,440) possible combinations that one can achieve from any given state of the 8 puzzle. This is nothing compared to the 15 puzzle, the 8 puzzle's larger counterpart, which has approximately 1.5*10^13 possible states.

These puzzles are a part of a set of problems which are referred to as "NP-Complete." NP-Complete means that it is easy to check a solution to see if it is correct, but it is diffult (computationally) to arrive to a correct solution. This set includes many other games like sodoku and battleships, as well as scientific and mathematic problems such as protein folding and vehicle routing. This is a huge topic of research in Computer Science today and anyone that can find an efficient way to solve NP problems would gain much glory and fame. This video provides a great overview of the topic.
